Building Python 2.7.4 on SLES 10 SP3

Posted on May 12, 2013

In this post, we’ll be building the latest revision of Python 2.7 on SLES 10 SP3. Please note that this procedure may also work on other versions of SLES and OpenSUSE too.

First, install the Python build dependencies:

1
2
3
sudo zypper -n install gcc
sudo zypper -n install sqlite-devel gdbm-devel zlib-devel openssl-devel ncurses-devel readline-devel
sudo zypper -n install tk-devel tcl-devel xorg-x11-devel

Download and extract Python:

1
2
3
wget http://www.python.org/ftp/python/2.7.4/Python-2.7.4.tgz
tar xvfz Python-2.7.4.tgz
cd Python-2.7.4/

Patch the sqlite module to work correctly with the sqlite libraries on SLES:

1
sed -i 's/sqlite3_int64/sqlite_int64/' ./Modules/_sqlite/*

Build Python with shared libraries (shared libraries ensure that Python works correctly with libraries like rubypython):

1
2
./configure --enable-shared
make

The following output is expected as the remaining modules are all pretty much useless and deprecated as per the official documentation:

Python build finished, but the necessary bits to build these modules were not found:
_bsddb             bsddb185           dl
imageop            sunaudiodev
To find the necessary bits, look in setup.py in detect_modules() for the module's name.

Finally, install Python to /usr/local:

1
sudo make install

Update the available libraries in the OS (so that the newly created libpython is available):

1
sudo /sbin/ldconfig -v

Clean up:

1
2
cd ..
rm -rfv Python-2.7.4

Install easy install (I could only get this working reliably by switching to root):

1
2
3
4
sudo su -
wget http://peak.telecommunity.com/dist/ez_setup.py
python ez_setup.py
rm -v ez_setup.py

Now install pip and distribute:

1
2
easy_install pip
pip install distribute

Drop out of root:

1
exit

Hope this helps! :)
